IntroductionÔÉÅ

üí° An Add-on file (extension: *.addon) is a ZIP archive with pre-defined folder- and filenames.

While an Add-on file will be created and modified with the Add-on Manager in most cases, it can also be unzipped/edited/zipped manually if necessary.

Content dataÔÉÅ

üí° Each ‚Äúcontent object‚Äù consists of a folder in one of the ‚Äúcontent type‚Äù (top level) folders.

  • A ‚Äúcontent object‚Äù is a single template/script/element/‚Ä¶ distributed via an Add-on.

  • Each ‚Äúcontent object‚Äù is represented by the content of a folder in the ‚Äúcontent type‚Äù folder matching the content type.

  • The exact format of the content representation depends on the ‚Äúcontent type‚Äù.

  • The name of the ‚Äúcontent object‚Äù is the same as its folder‚Äôs name.

  • For template-like objects, there is always a JSON file with administrative data present. Other types may vary.

Why is there a folder for every single element?ÔÉÅ

  • The Add-on Manager can handle the elements transparently in this way. Just the folder is dragged & dropped, visible etc., while the content is abstracted and can edited (e.g. in a graphical editor)

  • The content format specification of an element can easier be changed over time
